# Garden 콘텐츠 작성 스타일 가이드

이 디렉토리(`content/garden/`)에 새 글을 작성하거나 기존 글을 수정할 때 아래 규칙을 따라주세요.

---

## 톤 & 말투

- **"~했어요", "~해요", "~거든요", "~있어요"** 같은 친절한 존댓말을 사용해요.
- 1인칭 회고/에세이 톤으로 작성해요. 경험과 인사이트를 중심으로 풀어가요.
- 딱딱한 문어체(`~했다`, `~이다`)나 너무 캐주얼한 반말은 피해요.
- 독자에게 말을 거는 느낌으로 자연스럽게 써요.

### 톤 예시

```
(X) 앱 개발을 진행하면서 변경사항이 있을 때 재빌드 하는 것은 꽤 비효율적이다.
(O) 앱을 개발하다 보면, 작은 변경이 있을 때마다 재빌드하는 게 꽤 비효율적이에요.

(X) 클린 빌드를 기준으로 개발 환경과 프로덕션 환경의 빌드 속도를 비교해 보았다.
(O) 클린 빌드 기준으로 개발 환경과 프로덕션 환경의 빌드 속도를 비교해봤어요.

(X) 이 변화는 네이티브 개발자만의 이야기가 아니다.
(O) 이 변화는 네이티브 개발자만의 이야기가 아니에요.
```

## 글 구조

모든 글은 다음 구조를 따라요.

1. **"## 들어가며"** — 왜 이 주제에 관심을 가지게 되었는지, 어떤 상황이었는지 개인적 맥락을 담아요.
2. **본문 (H2 섹션들)** — 경험, 시행착오, 발견의 흐름으로 서사를 이어가요.
3. **"## 마치며"** — 핵심 인사이트 + 앞으로의 방향을 정리해요.
4. 섹션 사이에 `---` 구분선을 넣어요.

---

## 포맷팅 규칙

- **이모지 사용 금지**: 섹션 제목, 본문 어디에도 이모지를 넣지 않아요.
- **줄바꿈**: 문단 내 줄바꿈이 필요할 때는 `\`를 사용해요.
- **Bold**: 핵심 개념이나 결론을 강조할 때만 사용해요.
- **블록인용(>)**: 글의 핵심 인사이트나 테이크어웨이에만 사용해요.
- **코드 블록**: 코드 블록 앞에 반드시 왜 이 코드가 중요한지 맥락을 설명해요.

## Frontmatter

Garden 글은 `coverImage`를 필수로 포함해요.

```yaml
---
author: hogumachu
title: [한국어 제목]
subtitle: [한국어 부제]
coverImage: /images/covers/{slug}-cover.jpg
date: YYYY-MM-DD HH:MM
tags: [쉼표로 구분, 영어/한국어 혼용 가능]
published: true
---
```

작성 후에는 `content/garden/_cover-image-workflow.md`를 따라 커버 이미지를 생성하고,\
`public/images/covers/{slug}-cover.jpg` 파일 존재 여부와 frontmatter 경로 일치까지 확인해요.

### 작성 체크리스트 (필수)

- [ ] `content/garden/_cover-image-workflow.md` 기준으로 커버 이미지 생성
- [ ] `public/images/covers/{slug}-cover.jpg` 파일 존재 확인
- [ ] frontmatter `coverImage` 경로와 실제 파일 경로 일치 확인

## 글쓰기 원칙

- **경험 -> 시행착오 -> 인사이트** 흐름을 유지해요.
- 개인적 감정이나 판단을 자연스럽게 포함해요 ("느꼈어요", "체감했어요", "생각이 바뀌었어요").
- 정보만 나열하지 말고, "왜 이게 중요한지", "내가 왜 이 선택을 했는지" 서사를 입혀요.
- 기술적인 내용이라도 "경험을 공유하는 글" 느낌을 유지해요.
- 원래 내용과 의미를 변경하지 않아요. 같은 내용을 더 읽기 좋은 서사로 재구성하는 것이 목표예요.
